The deployment of models

The deployment is a essential phase in th lifecycle of a Machine Learning project.

It's about the transition between a developed model in development environment to the production environment.

  1. Model development: Before the deployment you must have a model developed and trained using historical data. At this moment you will select the correct algorithm, the feature engineering and the best model optmization.
  2. Validation and test: After the training the model is tested to make sure it will have the necessary performance in the production environment. It includes metrics like precision, recall, F1 Score and other.
  3. Deploy preparation:The model is converted to another formats like PMML (Predictive Model Markup Language) or ONNX (Open Neural Network Exchange).
  4. Infraestructure selection: The model can be deployed in local servers, on cloud (like AWS, Azure or Google Cloud) or even on edge devices (for IOT applications).
  5. Integration and model deployment: The model is integrated with the production system. For this step you could use APIs (Application Programming Interfaces) to make the link easier between the applications and services.
  6. Monitoring and maintenance: After the deployment, it's required the creation of monitoring tools to continuously detect problems, data drift or model degradation. This step will tell us if we need to re-train the model with newer data or even if we need to adjust the parameters to mantain good precision.
  7. Updates: The model deployment isn't static. Using feedbacks and following the monitoring indicators the model could be updated and imrpoved.

The deployment preparation

  1. Model optmization: It could be the shrink of the model size or even the hyperparameter optimization.
  2. Validation: Crossvaldation, use of new data to verify metrics like precision are examples.
  3. Convertion: The model should be converted to other formats depending of the production environment. It could be ONNX, TensorFlow SavedModels and others examples.
  4. Creation of Wrapper or API: It's created to make the comunication of the model and the aplications easier. It could be made using frameworks like Flask or FastAPI.
  5. Documentation: It's about the documentation of the model, like how it must be used, the limitations, versions and history of changes.
  6. Creation of Containers (optional): To make the deployment easier it's possible to create containers using tools like Docker.
  7. Security and compliance: It's the concern if the model is following the security and compliance laws like GDPR, HIPAA or LGPD.
  8. Rollout strategy: This step is about plan how the deployment will be like a gradual deployment to mitigate risks and even rollback plans if anything wents bad.
  9. Alerts and monitoring: Creation of monitoring systems to follow the model performance in production and send alerts when detect data drift or failures.

Containers and orquestration

Containers

Create a container is a way to pack a software and all that is necessary to his execution making it possible to execute the software independent of the environment.

It's possible to pack code, libraries, dependencies, environment variables.

A example of tool of this nature is "Docker".

Benefits of Containers

  • Allow the creation of a consistent environment for the applications independent of where it is running.
  • It can be moved easily from one environment to another.
  • Containers share the same core as the operational system making then lighter than traditional virtual machines.

Orquestration

The orquestration is the management of the life of the containers, specially in environments where there are many containers running at the same time.

The example of tools are "Kubernetes" and "Docker Swarm".

Benefits of Orquestration

  • It allow us to manage automatically the containers scale based on the workload.
  • It share automatically the requests between the containers to allow the efficient utilization of resources.
  • It's possible to auto-reset, change and kill containers with errors.
